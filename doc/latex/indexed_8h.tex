\subsection{Detailed Description}
\hyperlink{indexed_8h}{indexed.\+h} defines the indexed types and the lower level functions associated with their use. 

This header is modeled after cblas.\+h, and as such functions are prefixed with character sets describing the data types they operate upon. For example, the function {\ttfamily dfoo} would perform the function {\ttfamily foo} on {\ttfamily double} possibly returning a {\ttfamily double}.

If two character sets are prefixed, the first set of characters describes the output and the second the input type. For example, the function {\ttfamily dzbar} would perform the function {\ttfamily bar} on {\ttfamily double} {\ttfamily complex} and return a {\ttfamily double}.

Such character sets are listed as follows\+:
\begin{DoxyItemize}
\item d -\/ double ({\ttfamily double})
\item z -\/ complex double ({\ttfamily $\ast$void})
\item s -\/ float ({\ttfamily float})
\item c -\/ complex float ({\ttfamily $\ast$void})
\item di -\/ indexed double (\hyperlink{indexed_8h_a4f97de021a0bed1b84317670c9abd1bb}{double\+\_\+indexed})
\item zi -\/ indexed complex double (\hyperlink{indexed_8h_a1f2eabe291f8e4804c5d19c036a62292}{double\+\_\+complex\+\_\+indexed})
\item si -\/ indexed float (\hyperlink{indexed_8h_ad8c164750f0c6d25d7e5e2e15b463965}{float\+\_\+indexed})
\item ci -\/ indexed complex float (\hyperlink{indexed_8h_a87fb6b6c787120ead562dd4558003b73}{float\+\_\+complex\+\_\+indexed})
\item dm -\/ manually specified indexed double ({\ttfamily double}, {\ttfamily double})
\item zm -\/ manually specified indexed complex double ({\ttfamily double}, {\ttfamily double})
\item sm -\/ manually specified indexed float ({\ttfamily float}, {\ttfamily float})
\item cm -\/ manually specified indexed complex float ({\ttfamily float}, {\ttfamily float})
\end{DoxyItemize}

Throughout the library, complex types are specified via {\ttfamily $\ast$void} pointers. These routines will sometimes be suffixed by sub, to represent that a function has been made into a subroutine. This allows programmers to use whatever complex types they are already using, as long as the memory pointed to is of the form of two adjacent floating point types, the first and second representing real and imaginary components of the complex number.

The goal of using indexed types is to obtain either more accurate or reproducible summation of floating point numbers. Indexed types are composed of several adjacent bins...

The parameter {\ttfamily fold} describes how many bins are used in the indexed types supplied to a subroutine. The maximum value for this parameter can be set in config.\+h. If you are unsure of what value to use for , we recommend 3. Note that the {\ttfamily fold} of indexed types must be the same for all indexed types that interact with each other. Operations on more than one indexed type assume all indexed types being operated upon have the same {\ttfamily fold}. Note that the {\ttfamily fold} of an indexed type may not be changed once the type has been allocated. A common use case would be to set the value of {\ttfamily fold} as a global macro in your code and supply it to all indexed functions that you use.
